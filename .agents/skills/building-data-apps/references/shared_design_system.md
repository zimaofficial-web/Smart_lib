# Unified Data App Design System

Every data application built with this skill MUST adhere to these visual and functional principles to ensure a consistent, professional "SaaS-grade" experience.

## Design Principles
1. **Minimal Chrome**: Hide default framework UI (headers, footers, toolbars) to make the app feel standalone.
2. **Card-Based Layout**: Every logical piece of content (KPIs, Charts, Tables) MUST live in a bordered card with subtle shadows.
3. **Muted Color Palette**: 
   - Use a "Zinc" inspired palette (Grays/Zinc for text and borders).
   - Primary Accent: Blue (`#2563eb`).
4. **Semantic Coding**:
   - Success: Green
   - Error: Red
   - Warning: Amber
5. **Typography**:
   - Body/Headings: `DM Sans`
   - Data/Code: `JetBrains Mono`
6. **Spacing**: Use generous padding (at least `1.25rem` inside cards) and consistent gaps between elements.

## Color Palette (Zinc/shadcn style)
- **Background**: White (Light) / `#09090b` (Dark)
- **Card**: White (Light) / `#0c0c0f` (Dark)
- **Border**: `#e4e4e7` (Light) / `#1e1e24` (Dark)
- **Text**: `#09090b` (Light) / `#fafafa` (Dark)
- **Muted Text**: `#71717a` (Standard) / `#52525b` (Dark)
